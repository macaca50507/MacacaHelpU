POJ 3264

題目

使用 Sparse table 針對每一個 query 區間找出最大跟最小值相減就是答案
Sparse Table:
第一個維度 i 是長度( 2 ^ i )，第二個維度 j 是區間開頭位置
想法是透過將一個區間二分，再從前後兩個長度為 2 ^ (i - 1)的區間比較取出最大值
針對每一筆 query 對區間長度取 log 找出兩個可以涵蓋整個 query 的區間再找出最大/小

Buns

題目

這一題是多重背包的裸題，我自己覺得關鍵的地方是一開始的初始化要小心！
這一題順便幫我理清了我 dp 的一些概念，原本迴圈的迭代順序有時候都會搞不清楚

想法是使用前 i 種內餡，已經使用 j 克麵粉，所能產出的最大價值
那我們只需要考慮 "第 i 種" 做了幾個麵包 花費了多少麵粉就可以往前溯到 i - 1 項
初始化陷阱 : 如果麵粉有多出來，那值應該要是盡可能消耗光麵粉的 dp[0][i] 項 不是 0

New Year Table

題目

這一題學到的是 三角函數 arcsin arctan eps
1. pi = atan2( 0 , -1 ) 參數是( double y , double x )
2. acsin( double theta )
3. eps = 1e-8

在 double 運算底下會有誤差的情況產生 當我們在做不等式運算的時候
引入 eps 來幫我們處理誤差 經過移項之後基準點變為 0 與 eps 做比較
ex: ( x > y ) 變成 x - y > 0 > 0 的情況發生在 > eps
以 0 作為基準點 -eps ~ eps 都是視為 0
